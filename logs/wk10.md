## What I learned this week
[Key concepts — focus on understanding, not syntax]

- Making Decisions:
  - I learned how to use if, else if, and else to create different paths for the code to follow based on user input. 
    - ex: checking if a score is an "A" or a "B".

- **Strict Equality:** 
  - A key takeaway was the difference between == and ===. 
  - always use === because it checks both the value and the type, prevents weird bugs where the number 5 might accidentally equal the string "5".

- **Logical Operators:** 
  - I learned how to combine conditions using && (AND), || (OR), and ! (NOT). 
  - This is essential for things like form validation (e.g., checking if an email is present and a password is long enough).

- **Falsy Values:** 
  - Understanding that things like 0, "" (empty string), and null are "falsy" allows for much cleaner code when checking if a user has actually typed anything into a box.

- **Functions:** 
  - We practiced wrapping logic into reusable blocks. I learned the difference between a standard function declaration and an arrow function, which is a more modern, shorter way to write them.
